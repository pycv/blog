## 工作原理

### 1. Hugo 静态网站生成器

**核心原理**：
- Hugo 读取 Markdown 格式的文章（`content/`）
- 根据配置（`hugo.toml`）和主题（`themes/`）
- 生成完整的 HTML/CSS/JS 静态文件（`public/`）

**为什么选择 Hugo**：
- ⚡ 极快构建速度（毫秒级）
- 📝 Markdown 写作，简单高效
- 🎨 主题丰富，易于定制
- 🔒 静态文件，安全稳定

### 2. Git 版本控制

**作用**：
- 备份所有源代码和文章
- 追踪修改历史
- 支持多设备协作

**当前配置**：
```bash
远程仓库：git@github-pycv:pycv/blog.git
本地分支：main
SSH 密钥：~/.ssh/id_ed25519_github_asuspc_pycv
```

### 3. GitHub Actions 自动部署

**工作流程**：
```
推送代码到 GitHub
    ↓
触发 GitHub Actions (.github/workflows/deploy.yml)
    ↓
步骤 1: Checkout（拉取代码 + 子模块）
    ↓
步骤 2: Setup Hugo（安装 Hugo）
    ↓
步骤 3: Build（执行 hugo --minify）
    ↓
步骤 4: Upload（上传 public/ 目录）
    ↓
步骤 5: Deploy（部署到 GitHub Pages）
    ↓
完成！访问 https://pycv.github.io/blog/
```

**为什么需要 Actions**：
- ✅ 自动化：推送后自动构建部署
- ✅ 免费：GitHub 提供 CI/CD 服务
- ✅ 可靠：每次构建都有日志记录
- ✅ 多环境：支持不同分支和部署目标

## 日常维护

### 发布新文章

```bash
# 1. 创建新文章
hugo new content posts/my-new-post.md

# 2. 编辑文章（修改 draft: false）
# 使用你喜欢的编辑器编辑 content/posts/my-new-post.md

# 3. 本地预览（可选）
hugo server -D --port 9000
# 访问 http://localhost:9000/

# 4. 提交并推送
git add .
git commit -m "添加新文章：文章标题"
git push

# 5. 等待 1-2 分钟，自动部署完成
# 访问 https://pycv.github.io/blog/
```

### 修改网站配置

编辑 `hugo.toml` 文件：

```toml
# 修改网站标题
title = '你的新标题'

# 修改 baseURL（如果域名变更）
baseURL = 'https://你的新域名.com/'

# 添加新菜单项
[[menu.main]]
  identifier = "categories"
  name = "分类"
  url = "/categories/"
  weight = 4
```

修改后提交推送：
```bash
git add hugo.toml
git commit -m "更新网站配置"
git push
```

### 更换主题

```bash
# 1. 删除旧主题
git submodule deinit -f themes/旧主题名
rm -rf .git/modules/themes/旧主题名
git rm -f themes/旧主题名

# 2. 添加新主题（使用 Git 子模块）
git submodule add --depth=1 https://github.com/用户名/主题名.git themes/主题目录名

# 3. 更新 hugo.toml
# 修改：theme = '新主题目录名'

# 4. 提交推送
git add .
git commit -m "更换主题为 XXX"
git push
```

### 创建新的内容目录（Section）

以创建"摘录"目录为例：

```bash
# 1. 创建目录
mkdir -p content/excerpts

# 2. 创建索引页面（首页）
touch content/excerpts/_index.md

# 3. 编辑索引页面（_index.md）
+++
title = "摘录"
date = 2026-03-08
draft = false
description = "收藏的文章摘录和笔记"
+++

# 摘录

这里是收藏的文章摘录、阅读笔记和灵感记录。

# 4. 在该目录下创建内容
hugo new content excerpts/my-excerpt.md

# 5. 更新菜单配置（hugo.toml）
[[menu.main]]
  identifier = "excerpts"
  name = "摘录"
  url = "/excerpts/"
  weight = 3
  pre = "book"

# 6. 提交推送
git add .
git commit -m "添加摘录目录"
git push
```

**通用方法**：适用于创建任何新目录（如"作品集"、"资源"、"日记"等）

```bash
# 将上面步骤中的 excerpts 替换为你的目录名
# 将"摘录"替换为你的显示名称
# 调整 weight 值来控制菜单顺序
```

### 本地开发调试

```bash
# 启动开发服务器（支持热更新）
hugo server -D --port 9000

# 参数说明：
# -D: 显示草稿文章
# --port 9000: 指定端口（避免冲突）
# 文件修改后会自动刷新浏览器
```

### 构建静态网站

```bash
# 生产环境构建（压缩优化）
hugo --minify

# 构建后的文件在 public/ 目录
# 可以部署到任何静态网站托管服务
```

## 部署流程

### 首次部署

```bash
# 1. 创建 GitHub 仓库
# 访问 https://github.com/new
# 创建 blog 仓库（公开）

# 2. 配置 Git
cd D:\Code\blog
git init
git add .
git commit -m "初始化博客"

# 3. 连接远程仓库
git remote add origin git@github-pycv:pycv/blog.git

# 4. 推送代码
git branch -M main
git push -u origin main

# 5. 配置 GitHub Pages
# 访问 https://github.com/pycv/blog/settings/pages
# Source → 选择 "GitHub Actions"

# 6. 等待部署完成
# 访问 https://github.com/pycv/blog/actions
```

### 部署原理详解

**GitHub Actions 工作流文件**：`.github/workflows/deploy.yml`

```yaml
name: Deploy Hugo site to GitHub Pages

on:
  push:
    branches:
      - main          # 推送到 main 分支时触发
  workflow_dispatch:  # 支持手动触发

permissions:             # 权限配置
  contents: read        # 读取代码
  pages: write          # 写入 Pages
  id-token: write       # 部署令牌

jobs:
  build:                # 构建任务
    runs-on: ubuntu-latest
    steps:
      - Checkout       # 拉取代码
      - Setup Hugo     # 安装 Hugo
      - Build          # hugo --minify
      - Upload         # 上传构建产物

  deploy:               # 部署任务
    needs: build        # 依赖构建任务
    steps:
      - Deploy Pages    # 部署到 GitHub Pages
```

## 问题排查

### 1. 样式没有加载

**原因**：
- baseURL 配置错误
- 主题资源路径问题
- 浏览器缓存

**解决**：
```bash
# 检查 baseURL 是否正确
# hugo.toml: baseURL = 'https://pycv.github.io/blog/'

# 强制刷新浏览器
# Windows: Ctrl + Shift + R
# Mac: Cmd + Shift + R

# 重新构建
git add .
git commit -m "修复配置"
git push
```

### 2. GitHub Actions 失败

**检查步骤**：
1. 访问 https://github.com/pycv/blog/actions
2. 点击失败的工作流
3. 查看错误日志

**常见错误**：
- **YAML 语法错误**：检查 `.github/workflows/deploy.yml` 缩进
- **子模块未加载**：确认 `themes/` 是 Git 子模块
- **构建失败**：本地运行 `hugo --minify` 检查

### 3. 文章不显示

**检查**：
```bash
# 确认 draft: false
# front matter 应该是：
+++
title = "文章标题"
date = 2026-03-08
draft = false  # ⭐ 必须是 false
+++
```

### 4. SSH 连接失败

**测试连接**：
```bash
ssh -T git@github-pycv
# 应该显示：Hi pycv! You've successfully authenticated...
```

**检查配置**：
```bash
# 查看远程仓库地址
git remote -v
# 应该是：git@github-pycv:pycv/blog.git

# 查看 SSH 配置
cat ~/.ssh/config
# 应该包含 github-pycv 配置
```

2. **DNS 配置**：
   ```
   CNAME yourdomain.com → pycv.github.io
   CNAME www → pycv.github.io
   ```
3. **GitHub Pages 配置**：
   - Settings → Pages → Custom domain
   - 添加 yourdomain.com
4. **更新 baseURL**：
   ```toml
   baseURL = 'https://yourdomain.com/'
   ```

## 总结

### 核心理念

1. **内容为王**：专注于写作，技术自动处理
2. **版本控制**：所有内容都有历史记录
3. **自动化**：推送即部署，无需手动操作
4. **静态安全**：无数据库，无后端，速度快

### 维护流程

```
写作 → Git 提交 → 推送 → 自动构建 → 自动部署 → 完成
```

- ✅ 经常推送代码（自动备份）
- ✅ 写好 commit 信息（方便回溯）
- ✅ 本地测试后再推送
- ✅ 定期更新主题和 Hugo
- ✅ 备份 SSH 密钥
