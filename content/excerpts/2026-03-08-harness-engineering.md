- OpenAI 团队用 Codex（基于 GPT-5）从零构建真实产品，5 个月产出约 100 万行代码，全程工程师不直接写代码
- 当 Agent 搞不定时，不是自己上手，而是问"缺了什么工具、文档或能力"，让 Agent 自己去补
- 大文件 AGENTS.md 效果差：上下文是稀缺资源，所有东西都"重要"时什么都不重要，Agent 退化成局部模式匹配
- 解法是渐进式披露：把 AGENTS.md 精简到约 100 行，只充当目录和地图，指向结构化的 docs/ 目录
- 仓库是唯一的真相来源：Agent 访问不到的东西对它来说就不存在
- 偏好"无聊"的技术：可组合、API 稳定、在训练集中被充分覆盖的技术对 Agent 更友好
- 有时宁可自己重写也不用外部库：透明的内部代码比不透明的外部依赖有更高的"杠杆"
- 约束越严格，Agent 越高效：严格的架构约束让解空间更小，Agent 反而不容易跑偏
- 中央强制边界，局部允许自治：确定性 linter + LLM 审计 Agent 双管齐下
- 传统工程规范在高吞吐下变成阻碍：PR 生命周期要短，"纠错成本低，等待成本高"
- 技术债像高息贷款：持续小额偿还，远好过让它复利积累
- 未来软件工程师的核心竞争力不是写代码，而是设计"驾具"的能力——定义约束、构建反馈循环、管理文档结构
- 模型是马，驾具是缰绳。马越强壮，缰绳越重要
