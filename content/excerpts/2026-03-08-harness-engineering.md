## 核心观点

- **给 Agent 一张地图，而不是百科全书**：AGENTS.md 精简到约 100 行，只充当目录指向结构化 docs/，避免上下文稀释
- **仓库是唯一的真相来源**：Agent 访问不到的东西对它不存在，偏好"无聊"但稳定的技术
- **约束越严格，Agent 越高效**：固定依赖方向 + 确定性 linter + LLM 审计，中央强制边界、局部允许自治
- **模型是马，驾具是缰绳**：未来工程师的核心能力是设计"驾具"——定义约束、构建反馈循环、管理文档结构

## 补充要点

- OpenAI 用 Codex（GPT-5）5 个月产出 100 万行代码，全程工程师不直接写代码
- 当 Agent 搞不定时，不是自己上手，而是问"缺了什么"，让 Agent 自己补
- 有时宁可重写也不用外部库：透明内部代码 > 不透明外部依赖
- PR 生命周期要短，"纠错成本低，等待成本高"
- 技术债像高息贷款：持续小额偿还 > 复利后痛苦解决
- 人类品味只需捕获一次，然后在每行代码上持续强制执行
