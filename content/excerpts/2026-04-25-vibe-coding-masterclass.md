## 背景

Anthropic 研究员、《构建高效智能体》合著者 Erik Schluntz 分享了他在生产环境中实践 Vibe Coding 的完整方法论。他曾因手伤被迫将工作全权交给 Claude，在此过程中总结出一套可复制的工程化实践。网友 Movez 盛赞此演讲"胜过 100 门付费课程"。

## 核心观点

- **找到可验证的抽象层，而非逐行审查代码**：就像 CTO 靠验收测试管理工程师、PM 靠体验产品验证功能，开发者需要建立不读底层代码就能验证的检查点。Vibe Coding 的核心是"忘记代码，但始终关注产品"。
- **聚焦"叶子节点"策略**：将代码变更限制在不被其他模块依赖的末端功能上，允许在这些区域产生技术债。系统主干和底层架构仍需人工严密保护。随着模型能力提升，可信任 AI 接管的层级正在向下延伸。
- **把自己当 Claude 的产品经理**：不要问 Claude 能为你做什么，要问你能为 Claude 做什么。Erik 的标准流程是花 15-20 分钟让 AI 先探索代码库、查找相关文件、制定计划，再汇总成清晰的提示词执行。
- **22000 行代码的生产级合并**：Anthropic 内部在 RL 代码库中成功合并了 22000 行 AI 生成代码，核心策略：数天前期规划 + 叶子节点限定修改范围 + 核心逻辑人工审查 + 长时间压力测试。两周工作量压缩到 1 天。

## 补充要点

- **TDD 是 Vibe Coding 的安全网**：强制只写极简的端到端测试（快乐路径 + 2 个错误场景），确保人类能看懂。测试是 Vibe Coding 中唯一需要亲自审查的代码。
- **上下文压缩时机**：在"人类程序员会停下来吃午饭"的自然断点处进行 Compact。先让 Claude 把计划写入文档，再压缩，将 10 万 Token 压到几千。
- **工具链协同**：Claude Code 负责主要修改和框架搭建，Cursor 用于精确行级修改和收尾修复。面对陌生代码库，先用 Claude Code 探索建立全局视图。
- **关于安全**：做好 PM 是根本——需要懂行，知道什么是危险的。对生产系统必须问出正确的问题引导 AI。期待涌现"数学证明级别正确"的产品和沙盒化前端开发环境。
- **指数级增长的含义**：不是"好两倍"，而是"好一百万倍"。从 KB 到 TB 的存储跃迁就是先例。模型能力提升的速度远超想象。
