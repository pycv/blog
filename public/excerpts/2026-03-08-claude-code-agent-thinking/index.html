Claude Code 工程师：我们花了一年，才学会像 Agent 一样思考

2026-03-08 1 minute read

本文由 AI 自动摘录整理，原始链接：Claude Code 工程师：我们花了一年，才学会像 Agent 一样思考

推荐指数：8/10（推荐读原文）

背景

Claude Code 核心开发者 Thariq 写了一篇长推「Lessons from Building Claude Code: Seeing like an Agent」，分享了他们一年多来设计 Agent 工具的心得——不是发布新功能，而是讲怎么想明白该给 AI 配什么工具这件事。

核心观点

  • 工具能力上限取决于使用者能力上限：给不会编程的人一台超级电脑，不如给他计算器；同理，给 Claude 配一堆它用不好的工具，不如配几个它真正懂得调用的工具
  • 一个工具一个语义，混合意图会让模型困惑：AskUserQuestion 工具迭代了三次才成功——失败原因分别是语义冲突、自由文本不稳定、模型不知道何时调用
  • 模型能力进化，工具设计需要跟着迭代：TodoWrite 曾经必要，但后来反而束缚了更强的模型；今天正确的约束，明天可能是枷锁
  • 渐进式披露：不加工具也能加能力：通过「Context on Demand」让 Agent 需要时再拉取上下文，而不是把文档全塞进系统提示

补充要点

  • 学会"像 Agent 一样看世界"：读它的输出、做实验、持续观察，而不是假设它能或不能做什么
  • 工具多了有代价：每多一个选项，模型就多一个决策负担，性能和稳定性都会受影响
  • Claude Code Guide 子 Agent：专门回答关于 Claude Code 自身的问题，有专门优化的搜索指令
  • Grep 工具替代 RAG：让 Claude 自己搜索代码库，主动构建自己需要的上下文
  • 工具设计是艺术不是科学：高度依赖模型、目标、环境三个变量，唯一可靠的路径是持续实验
